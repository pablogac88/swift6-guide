Swift 6.0

Actor · Strict Concurrency

Los data races pasan de ser crashes en producción a errores en tiempo de compilación. El actor reemplaza a DispatchQueue con garantías verificadas por el compilador.

El problema — DispatchQueue como acuerdo de honor

En Swift 5, proteger estado mutable compartido era responsabilidad tuya. Usabas una DispatchQueue y rezabas para no olvidarte de ella en ningún acceso. El compilador no sabía nada de queues y no podía ayudarte.

Swift 5.x — el problema

class ImageCache {
    private var storage: [URL: Data] = [:]
    private let queue = DispatchQueue(
        label: "image-cache",
        attributes: .concurrent
    )

    func data(for url: URL) -> Data? {
        queue.sync { storage[url] }
    }

    func insert(_ data: Data, for url: URL) {
        queue.async(flags: .barrier) {
            self.storage[url] = data
        }
    }

    // Cualquier función nueva que olvidara la queue...
    func invalidate(url: URL) {
        storage.removeValue(forKey: url) // ← race silencioso
        // El compilador no dice nada. El bug aparece a las 3am.
    }
}

La solución — actor

Un actor serializa automáticamente todos los accesos a su estado. No hay queue que recordar. El compilador obliga a usar await en cada acceso externo — si intentas saltártelo, es un error de compilación, no un crash en producción.

Swift 6.x — la solución

actor ImageCache {
    private var storage: [URL: Data] = [:]

    func data(for url: URL) -> Data? {
        storage[url]
    }

    func insert(_ data: Data, for url: URL) {
        storage[url] = data
    }

    func invalidate(url: URL) {
        storage.removeValue(forKey: url) // ✓ seguro, estamos dentro del actor
    }
}

// En el sitio de llamada, await hace explícito el cruce de contexto
let cache = ImageCache()
await cache.insert(data, for: url)
let cached = await cache.data(for: url)
// Sin await → error de compilación. Imposible olvidarse.

La regla: El actor hace lo mismo que DispatchQueue pero con garantías en tiempo de compilación. El await no es solo sintaxis — es el compilador diciéndote "aquí cruzas al contexto del actor".

¿Por qué importa para iOS?

En apps iOS, los singletons, view models, routers y managers son los candidatos naturales. En lugar de DispatchQueue.main.async para tapar problemas de aislamiento, marca el tipo con @MainActor o conviértelo en actor. El compilador te guiará.
