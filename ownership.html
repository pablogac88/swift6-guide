Swift 6.0

Ownership · ~Copyable

Tipos que no se pueden duplicar. El compilador garantiza que un recurso se usa exactamente una vez.

Antes — todo era copiable, los recursos se podían duplicar

En Swift 5, todos los tipos son copiables por defecto. No había forma de modelar un recurso que solo puede usarse una vez. Los file handles, tokens y sockets podían duplicarse accidentalmente — el compilador no podía detectarlo.

Swift 5.x — el problema

// Swift 5 — nada impide usar el token dos veces
struct AccessToken {
    let value: String

    func consume() {
        print("Token usado: \(value)")
        // No hay ninguna garantía de que esto no se llame dos veces.
    }
}

let token = AccessToken(value: "abc123")
token.consume()   // ok
token.consume()   // doble uso → error lógico, el compilador calla
let copy = token
copy.consume()    // triple uso → nadie lo detecta
// Los bugs aparecen en runtime: doble cierre, doble pago, token expirado...

Ahora — ~Copyable + consuming

~Copyable suprime la copia. Una vez que consumes el valor, el compilador sabe que ya no existe. Intentar usarlo de nuevo es un error de compilación.

Swift 6.x — la solución

// ~Copyable: este tipo no se puede duplicar
struct AccessToken: ~Copyable {
    let value: String

    // consuming: la función toma ownership y destruye el valor al salir
    consuming func consume() {
        print("Token usado: \(value)")
        // Al salir, self se destruye. No hay doble uso posible.
    }
}

let token = AccessToken(value: "abc123")
token.consume()   // ✓ el token pasa a la función y se destruye

// token.consume()   ← ERROR de compilación
//                   'token' used after consume
// let copy = token  ← ERROR de compilación
//                   cannot copy noncopyable type 'AccessToken'

La regla: El bug de "usar un recurso dos veces" pasa de ser un error de runtime (crash a las 3am) a ser un error de compilación (antes de ejecutar ni una línea).

Protocolos con ~Copyable

~Copyable en un protocolo no contagia a los conformantes — solo abre la puerta para que tipos no copiables puedan conformarlo. Cada tipo decide individualmente si es copiable o no.

Swift 6.x — protocolos

// ~Copyable en el protocolo = "acepto copiables Y no copiables"
// Sin ~Copyable = "solo acepto copiables" (Coffee no podría conformarlo)
protocol Drinkable: ~Copyable {
    consuming func use()
}

struct Coffee: Drinkable, ~Copyable {  // no copiable
    let brand: String
    consuming func use() { print("☕ \(brand)") }
}

struct Water: Drinkable {  // copiable (por defecto)
    consuming func use() { print("💧") }
}

// consuming some Drinkable & ~Copyable = acepta ambos
func drink(item: consuming some Drinkable & ~Copyable) {
    item.use()
}
Palabra claveSignificado
~Copyableel tipo no se puede duplicar, solo transferir
consumingla función toma ownership, el caller ya no puede usar el valor
borrowingla función lee el valor sin tomarlo, el caller lo recupera
