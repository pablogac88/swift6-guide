Swift 6.2

Approachable Concurrency · @concurrent

@concurrent declara explícitamente dónde corre una función. Named tasks son visibles en LLDB e Instruments.

El problema — nonisolated async era ambiguo

En Swift 6.0, cuando marcabas una función async como nonisolated, no quedaba claro en qué executor iba a correr. Dependía del contexto del caller de formas no siempre obvias. Y las tasks eran anónimas — invisibles en Instruments.

Swift 6.0 — el problema

// Swift 6.0 — nonisolated async: ¿dónde corre esto exactamente?
nonisolated func fetchData(at url: URL) async throws -> Data {
    // ¿Corre en el executor del caller? ¿En el pool global?
    // Dependía del contexto. Era fácil hacer suposiciones incorrectas.
    let (data, _) = try await URLSession.shared.data(from: url)
    return data
}

// Tasks anónimas — invisibles en herramientas de debugging
let task = Task {
    try await fetchData(at: url)
    // En Instruments: "Task" sin nombre. Con decenas de tasks,
    // imposible saber cuál es cuál.
}

Ahora — @concurrent explícito + named tasks

@concurrent declara sin ambigüedad que la función siempre corre en el executor global concurrente, independientemente del caller. Task(name:) da nombre a las tasks para debugging y profiling.

Swift 6.2 — la solución

@MainActor
struct ImageStore {
    static var cache: [URL: Data] = [:]

    // Sin anotación: hereda el aislamiento del caller (@MainActor aquí)
    static func create(from url: URL) async throws -> Data {
        if let cached = cache[url] { return cached }
        let data = try await fetchData(at: url)  // sale al background
        cache[url] = data
        return data
    }

    // @concurrent: SIEMPRE en el pool global, nunca en el MainActor
    // Antes: nonisolated con semántica incierta
    // Ahora: explícito, sin ambigüedad
    @concurrent
    static func fetchData(at url: URL) async throws -> Data {
        let (data, _) = try await URLSession.shared.data(from: url)
        return data
    }
}

// Named tasks: nombre visible en LLDB, Instruments y crash reports
let prefetch = Task(name: "prefetch-home-feed") {
    try await HomeRepository.shared.prefetch()
}
// En Instruments → la task aparece etiquetada "prefetch-home-feed"
// En LLDB: thread info → "Task: prefetch-home-feed"

La regla: La regla de Swift 6.2 es simple — sin anotación hereda el contexto del caller, @concurrent siempre va al pool global, @MainActor siempre al hilo principal.
