Swift 6.2

@MainActor por defecto · Aislamiento implícito

En Swift 6.2 puedes declarar que todo tu módulo corre en el MainActor por defecto, y solo optar a salir con @concurrent o nonisolated. El modelo que ya usas en UIKit/SwiftUI, ahora explícito y verificado.

Antes — anotar @MainActor manualmente en todo

En Swift 6.0 el compilador asumía que las funciones no estaban aisladas (nonisolated por defecto). Esto forzaba a anotar manualmente con @MainActor todo lo que tocaba UI, generando mucho ruido repetitivo en los view models.

Swift 6.0 — el problema

// Swift 6.0 — anotar @MainActor manualmente en todo
@MainActor
class ProfileViewModel: ObservableObject {
    @MainActor func loadUser() async { ... }
    @MainActor func updateAvatar() async { ... }
    // repetitivo — y si olvidas uno, error de compilación en el sitio de llamada
}

Ahora — aislamiento por defecto del módulo

Con .defaultIsolation(MainActor.self) en Package.swift, todo el módulo está en MainActor implícitamente. Para trabajo en background, optas a salir con @concurrent. El modelo mental correcto para apps iOS: UI en main, trabajo pesado opt-in.

Swift 6.2 — la solución

// Swift 6.2 — con aislamiento por defecto en el módulo
// En Package.swift:
// swiftSettings: [.defaultIsolation(MainActor.self)]

// Ahora todo está en @MainActor por defecto
class ProfileViewModel {
    func loadUser() async { ... }      // implícitamente @MainActor
    func updateAvatar() async { ... }  // implícitamente @MainActor

    // Para trabajo en background, opt-out explícito:
    @concurrent
    func fetchFromNetwork() async -> Data { ... }
}

La regla: Activa esto en Package.swift con .defaultIsolation(MainActor.self). Es un cambio de modelo: en lugar de "nonisolated hasta que demuestres lo contrario", es "MainActor hasta que necesites salir". Mucho más cercano a cómo piensa un developer iOS.

El modelo mental para apps iOS

La mayoría del código de una app iOS toca UI directa o indirectamente. Hacer MainActor el default y optar a salir para trabajo pesado es la inversión correcta.
