Swift 6.1

nonisolated extension

Una sola declaración para toda la extensión en lugar de repetir nonisolated en cada método.

El problema — @MainActor contaminaba todo el tipo

En Swift 6.0, si un tipo era @MainActor, el compilador asumía que todos sus métodos — incluso los de protocolos como Equatable — solo podían ejecutarse en el hilo principal. Comparar dos IDs no tiene nada que ver con la UI, pero el compilador lo exigía igual.

Swift 6.0 — el problema

// Swift 6.0 — @MainActor en el tipo contamina todos los métodos
@MainActor
struct UserViewState {
    let id: Int
    let name: String
}

// ERROR: para llamar a == hay que estar en el @MainActor
// → await obligatorio para una comparación trivial
extension UserViewState: Equatable {
    static func ==(lhs: Self, rhs: Self) -> Bool {
        lhs.id == rhs.id
    }
}

// Solución en Swift 6.0: nonisolated en cada método por separado
extension UserViewState: CustomStringConvertible, Equatable {
    nonisolated var description: String { "id: \(id)" }
    nonisolated static func ==(lhs: Self, rhs: Self) -> Bool { lhs.id == rhs.id }
    nonisolated var isValid: Bool { id > 0 }
    // Repetitivo si tienes muchos miembros
}

Ahora — nonisolated en la extensión completa

nonisolated extension aplica nonisolated a todos los miembros a la vez. El resultado es idéntico — el compilador lo trata igual — pero sin repetición.

Swift 6.1 — la solución

@MainActor
struct UserViewState {
    let id: Int
    let name: String
}

// Swift 6.1 — una sola declaración para toda la extensión
nonisolated extension UserViewState: CustomStringConvertible, Equatable {
    var description: String { "UserViewState(id: \(id), name: \(name))" }
    static func ==(lhs: Self, rhs: Self) -> Bool { lhs.id == rhs.id }
    var isValid: Bool { id > 0 }
    // Todos los miembros son nonisolated — sin repetir la palabra
}

// Ahora se pueden llamar desde cualquier contexto sin await:
let a = UserViewState(id: 1, name: "Pablo")
let b = UserViewState(id: 1, name: "García")
print(a == b)          // ✓ sin await
print(a.description)   // ✓ sin await

La regla: nonisolated extension es la forma de decirle al compilador "esta parte del tipo es lógica pura, no UI — déjame usarla desde cualquier sitio sin obligarme al hilo principal".

¿Cuándo es seguro?

El compilador verifica automáticamente que lo que lees sea accesible desde fuera del actor. La regla real es cuándo puede haber una escritura simultánea:

CasoSeguro
struct + letSí — copia inmutable, nadie puede cambiarla
struct + varSí — cada contexto tiene su propia copia
class + letSí — instancia compartida pero inmutable
class + varNo — instancia compartida y mutable → race posible
