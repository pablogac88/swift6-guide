Swift 6.1

Task groups · Trailing commas

El compilador infiere el tipo en task groups. Trailing comma válida en argumentos, tuplas y closures.

Antes — el tipo era obligatorio y la coma final daba error

withTaskGroup requería especificar el tipo de las child tasks con of:. Era ruido puro cuando el tipo era obvio. Y una coma al final de una lista de argumentos era error de sintaxis.

Swift 6.0 — el problema

// Swift 6.0 — of: Message.self obligatorio
let messages = await withTaskGroup(of: Message.self) { group in
    //                             ^^^^^^^^^^^^^^^^
    //                             Sin esto → error de compilación.
    //                             En tipos complejos resultaba muy verboso:
    //   withTaskGroup(of: (String, [Article], UserProfile).self)
    for id in ids {
        group.addTask { await downloadMessage(for: id) }
    }
    var result: [Message] = []
    for await message in group { result.append(message) }
    return result
}

// Coma final → error de sintaxis en Swift 5.x / 6.0:
numbers.split(
    separator: 0,
    maxSplits: 1   // ← no podías añadir la coma aquí sin error
)

Ahora — inferencia de tipo y trailing commas

Swift 6.1 infiere el tipo de las child tasks desde el closure. of: ya no hace falta. Y las trailing commas son válidas en argumentos, tuplas y closure capture lists.

Swift 6.1 — la solución

// Swift 6.1 — sin of:, el tipo se infiere del closure
let messages = await withTaskGroup { group in
    for id in ids {
        group.addTask { await downloadMessage(for: id) }
    }
    var result: [Message] = []
    for await message in group { result.append(message) }
    return result
}

// Trailing comma en argumentos — válida en Swift 6.1
numbers.split(
    separator: 0,
    maxSplits: 1,   // ← trailing comma válida
)

// Trailing comma en tupla — válida en Swift 6.1
let info = (
    user.id,
    user.name,      // ← trailing comma válida
)

La regla: Las trailing commas parecen pequeñas, pero hacen que añadir un argumento en un diff solo toque una línea, no dos. Macros y generación de código lo agradecen especialmente.
