Swift 6.0

Atomic<T> · Synchronization

Operaciones atómicas en Swift puro. Para contadores, flags y referencias que necesitan acceso seguro sin un lock completo.

Antes — sin atómicos nativos

Swift no tenía atómicos nativos. Para algo tan simple como incrementar un contador de forma thread-safe necesitabas OSAtomicAdd32 de C, una DispatchQueue serial, o un NSLock. Todo era verboso, o dependía de APIs de bajo nivel, o inseguro.

Swift 5.x — el problema

// Swift 5.x — contador thread-safe con DispatchQueue
final class RequestCounter {
    private var _count = 0
    private let queue = DispatchQueue(label: "counter")

    func increment() {
        queue.sync { _count += 1 }
    }

    var count: Int {
        queue.sync { _count }
    }
}

Ahora — Atomic<T> del módulo Synchronization

El módulo Synchronization trae Atomic<Int> con operaciones de hardware. .load(ordering:), .store(_:ordering:), .add(_:ordering:). Más ligero que Mutex para operaciones simples porque no necesita un lock.

Swift 6.0 — la solución

import Synchronization

// Swift 6.0 — Atomic del módulo Synchronization
final class RequestCounter: Sendable {
    // Atomic almacena el valor con garantías de hardware
    private let _count = Atomic<Int>(0)

    func increment() {
        // .relaxed para contadores donde el orden exacto no importa
        _count.add(1, ordering: .relaxed)
    }

    var count: Int {
        _count.load(ordering: .acquiring)
    }
}

// Uso natural, sin queues ni locks visibles
let counter = RequestCounter()
// En varios Tasks concurrentes:
counter.increment()
print(counter.count)  // siempre correcto

Mutex vs Atomic: Usa Mutex para operaciones compuestas (leer-modificar-escribir con lógica propia). Usa Atomic para operaciones simples como incrementar, comparar o intercambiar. Atomic es más rápido pero solo cubre tipos básicos: Int, Bool, punteros.

Cuándo usar cada uno

La elección depende de la complejidad de la operación. Para un contador, Atomic. Para un diccionario que lees y modificas con condiciones, Mutex.

HerramientaCuándo
Atomic<T>operaciones simples: incrementar, comparar, intercambiar. Tipos básicos. Máximo rendimiento.
Mutex<T>operaciones compuestas con lógica propia, cualquier tipo, incluyendo colecciones.
