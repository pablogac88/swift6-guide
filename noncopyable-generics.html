Swift 6.3

~Copyable en genéricos · Sin restricciones

En Swift 6.0, ~Copyable tenía fricciones con genéricos y protocolos. Swift 6.3 lo generaliza: tipos non-copyable pueden participar en código genérico sin workarounds.

Antes — ~Copyable se "perdía" en código genérico

En Swift 6.0, no podías usar ~Copyable como parámetro de tipo genérico directamente. Los protocolos que conformaban ~Copyable tenían limitaciones en cómo podías usarlos en funciones genéricas. Útil en tipos concretos, frustrante en abstracciones.

Swift 6.0 — el problema

// Swift 6.0 — limitaciones con ~Copyable en genéricos
// Esto NO compilaba en 6.0:
func transfer<T: ~Copyable>(_ value: consuming T) -> T {
    value  // error: ~Copyable en genérico no soportado completamente
}

Ahora — ~Copyable en genéricos funciona

Las funciones genéricas pueden declarar <T: ~Copyable>. Los protocolos pueden conformar ~Copyable. Optional, Result y los algoritmos de la librería estándar entienden tipos non-copyable.

Swift 6.3 — la solución

// Swift 6.3 — ~Copyable en genéricos funciona
func transfer<T: ~Copyable>(_ value: consuming T) -> T {
    value  // ✓
}

// Protocolo con ~Copyable generalizado
protocol Resource: ~Copyable {
    consuming func release()
}

struct GPUBuffer: Resource, ~Copyable {
    let id: UInt32
    consuming func release() {
        // liberar recurso GPU exactamente una vez
        print("Liberando buffer GPU \(id)")
    }
}

// Función genérica que acepta cualquier Resource ~Copyable
func useAndRelease<R: Resource & ~Copyable>(_ resource: consuming R) {
    resource.release()
    // El compilador verifica que no usamos `resource` después
}

// La librería estándar también:
// Optional<T: ~Copyable>, Result<T: ~Copyable, E>
let buffer: GPUBuffer? = GPUBuffer(id: 42)
if let b = consume buffer {
    useAndRelease(b)
}

La regla: En Swift 6.0, ~Copyable era útil en tipos concretos pero se "perdía" al entrar en código genérico. Swift 6.3 cierra esa brecha. Especialmente importante para tipos que representan recursos únicos: handles de fichero, conexiones, buffers GPU.

Tipos que se benefician de ~Copyable en genéricos

Los recursos que deben liberarse exactamente una vez son los candidatos naturales. ~Copyable en genéricos permite escribir abstracciones sobre ellos sin sacrificar las garantías.

Tipo de recursoGarantía con ~Copyable
FileDescriptorse cierra exactamente una vez, no se puede duplicar accidentalmente
GPUBufferse libera exactamente una vez, sin double-free
NetworkConnectionse cierra exactamente una vez, sin múltiples cierres
CryptoKeyno se puede copiar accidentalmente a contextos inseguros
