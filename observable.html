Swift 6.2

@Observable · Observations

De ObservableObject + @Published + Cancellable a una sola macro. Sin Combine.

Antes — tres conceptos para un modelo observable

Para hacer un objeto observable en SwiftUI necesitabas: ObservableObject, @Published en cada propiedad que querías observar, y Cancellable para gestionar subscripciones en UIKit. Olvidarse de @Published en una propiedad era un bug silencioso.

Swift 5 / iOS 13-16 — el problema

// Swift 5 / iOS 13-16 — con Combine
import Combine

final class CartModel: ObservableObject {
    @Published var itemCount = 0    // ← @Published en CADA propiedad
    @Published var total = 0.0      // ← sin @Published → SwiftUI no reacciona
    var internalState = false       // ← sin @Published, intencional
    // Si olvidas @Published en una propiedad que debería serlo → bug silencioso
}

// En UIKit — subscripción manual con Cancellable
var cancellables = Set<AnyCancellable>()

model.$itemCount
    .sink { [weak self] count in
        self?.updateBadge(count)
    }
    .store(in: &cancellables)  // ← sin esto → memory leak garantizado
// Además: no funciona con structs, solo con clases

Ahora — @Observable

Una sola macro convierte todas las propiedades stored en observables automáticamente. Sin Combine, sin @Published, sin Cancellable. SwiftUI re-renderiza solo las vistas que leen propiedades que han cambiado.

Swift 5.9+ / iOS 17+ — la solución

// Swift 5.9+ / iOS 17+ — con @Observable
import Observation

@Observable
final class CartModel {
    // Todas las propiedades stored son observables automáticamente
    var itemCount = 0
    var total = 0.0
    // Sin @Published, sin ObservableObject, sin Combine
}

// En SwiftUI — sin cambios, simplemente funciona
struct CartView: View {
    let cart: CartModel

    var body: some View {
        // SwiftUI observa exactamente las propiedades que esta vista LEE.
        // Si cambia itemCount pero esta vista no lo usa → no se re-renderiza.
        Text("Total: \(cart.total, format: .currency(code: "EUR"))")
    }
}

// En Swift 6.2 — Observations como AsyncSequence (macOS 26+)
// Stream continuo de cambios, transaccional:
// for await snapshot in Observations { (cart.itemCount, cart.total) } {
//     print("Items: \(snapshot.0), total: \(snapshot.1)")
// }
// Si cambian los dos en el mismo ciclo → una sola emisión (transaccional)

La regla: La diferencia clave en SwiftUI: @Published notificaba que "algo cambió en el modelo" y SwiftUI re-renderizaba todo. @Observable rastrea qué propiedades lee cada vista y solo re-renderiza las afectadas.

Lo que desaparece con @Observable

ObservableObject en la declaración del modelo. @Published en cada propiedad. AnyCancellable y .store(in: &cancellables) en UIKit. .sink { } para observar cambios. Las subscripciones manuales. Funciona con structs, no solo clases.
