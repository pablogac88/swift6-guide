Swift 6.0

BitwiseCopyable · Copia sin ARC

Un protocolo que marca tipos que se pueden copiar bit a bit, sin llamadas a ARC ni destructores. El compilador los trata como valores C planos.

Antes — Swift no distinguía entre tipos que necesitan ARC y los que no

Swift no distinguía entre tipos que necesitan ARC y tipos que no. Un struct con solo Int y Bool aún podía pasar por rutas de código más lentas. Para interop con C o código de alto rendimiento, no había forma de garantizar en el sistema de tipos que un tipo era "plano".

Ejemplo

// BitwiseCopyable: el tipo es tan simple que se puede copiar con memcpy
struct Pixel: BitwiseCopyable {
    var r: UInt8
    var g: UInt8
    var b: UInt8
    var a: UInt8
}

// struct con solo tipos BitwiseCopyable → automáticamente BitwiseCopyable
struct Transform: BitwiseCopyable {
    var x: Float
    var y: Float
    var scale: Float
}

// String NO es BitwiseCopyable (tiene buffer en heap)
// class NO es BitwiseCopyable (tiene conteo de referencias)
// struct con String tampoco lo es

// Uso en funciones genéricas de alto rendimiento
func processBatch<T: BitwiseCopyable>(_ items: [T]) {
    // El compilador sabe que puede copiar T con memcpy
    // Sin llamadas a retain/release en el loop
    for item in items {
        _ = item
    }
}

La regla: Los tipos que conforman BitwiseCopyable automáticamente son: Int, Float, Bool, Double, UInt8... y cualquier struct compuesto solo de ellos. Útil sobre todo al trabajar con Metal, CoreAudio, o buffers de red donde procesas millones de elementos.

Qué puede ser BitwiseCopyable y qué no

La regla es simple: si el tipo no tiene referencias de heap ni necesita cleanup al ser destruido, puede ser BitwiseCopyable.

TipoBitwiseCopyable
Int, Float, Bool, Double✓ automáticamente
struct solo con Int/Float/Bool✓ automáticamente
String✗ tiene buffer en heap
Array, Dictionary✗ tienen buffer en heap
class (cualquiera)✗ tiene conteo de referencias
struct con String✗ hereda la limitación
